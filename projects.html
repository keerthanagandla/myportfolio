Saving Seconds, Saving Lives: Geospatial Analysis of Traffic Collision Hotspots and Hospital Accessibility in Los Angeles

This data science project analyzes large-scale Los Angeles traffic collision data to identify accident hotspots, evaluate hospital accessibility, and develop machine learning models that predict crash severity.

Project Overview

Using geospatial analytics and machine learning, the project integrates traffic collision records with hospital datasets to understand how location, behavior, and infrastructure influence crash severity.

Key Objectives

  • Identify traffic collision hotspots across Los Angeles
  • Measure proximity of accidents to hospitals
  • Analyze temporal and behavioral accident patterns
  • Predict crash severity using machine learning models

Tools & Technologies

  • Python
  • Pandas & NumPy
  • LightGBM & CatBoost
  • DBSCAN clustering
  • Geospatial analytics

Visual Analytics of Netflix Movies and TV Shows Using Tableau

This project explores Netflix’s global catalog using Tableau dashboards to analyze content distribution, genre patterns, maturity ratings, and production trends across countries.

Project Overview

Netflix hosts thousands of movies and TV shows produced across many countries and genres. Understanding how this content is distributed helps reveal trends in global media production and audience targeting. This project transforms raw Netflix metadata into interactive visual dashboards that highlight key insights such as genre popularity, rating distributions, country-level production, and growth trends over time.

Dataset

The dataset contains metadata for 8,807 Netflix titles including both Movies and TV Shows.

  • Type (Movie or TV Show)
  • Title
  • Director
  • Country of production
  • Maturity rating
  • Release year
  • Date added to Netflix
  • Content duration
  • Genre categories

The dataset spans content from more than 100 countries and several decades of releases, making it suitable for analyzing global streaming trends and Netflix’s evolving catalog.

Data Preparation

  • Handled missing values in key fields such as director, rating, and country
  • Split multi-valued genre fields for more accurate genre analysis
  • Standardized date formats for time-series visualizations
  • Removed redundant or inconsistent fields
  • Cleaned text inconsistencies and formatting issues

Key Visualizations

  • Movies vs TV Shows distribution
  • Rating distribution across content types
  • Growth of Netflix titles by year
  • Top 10 genres on Netflix
  • Top producing countries
  • Top directors contributing content

Key Insights

  • Movies represent about 68% of Netflix’s catalog while TV shows account for about 32%
  • TV-MA and TV-14 dominate maturity ratings
  • Netflix content expanded rapidly between 2015 and 2019
  • The United States, India, and the United Kingdom contribute the most titles
  • International Movies, Dramas, and Comedies are the most common genres

Loan Eligibility Prediction Through Predictive Modeling

This project applies machine learning techniques to predict loan approval outcomes for Sunrise Financial Services using applicant demographic and financial information.

Project Overview

Financial institutions must evaluate loan applications quickly and accurately to reduce risk and improve customer experience. Manual evaluation can be slow and may introduce bias. This project develops predictive machine learning models that automatically determine whether a loan application should be approved based on applicant attributes such as income, education, employment status, and credit history.

Dataset

The dataset contains customer application information collected from Sunrise Financial Services through an online loan application system.

  • Training dataset with 16 attributes
  • Testing dataset with 15 attributes
  • Target variable: Loan_Status (Approved / Not Approved)
  • Applicant demographics and financial indicators

Key Features

  • Gender
  • Marital Status
  • Education
  • Employment Status
  • Applicant Income
  • Co-applicant Income
  • Loan Amount
  • Loan Term
  • Credit History
  • Property Area

Data Preprocessing

  • Handled missing values using median and mode imputation
  • Treated outliers using the IQR method
  • Encoded categorical variables for machine learning
  • Balanced dataset using RandomOverSampler

Feature Engineering

  • Created TotalIncome by combining applicant and co-applicant income
  • Developed IncomePerPerson to measure financial burden
  • Introduced LoanToIncomeRatio to evaluate repayment capability
  • Converted loan term from months into years

Machine Learning Models

  • Logistic Regression
  • Random Forest
  • Decision Tree
  • XGBoost

Model Performance

  • Random Forest: ~91% accuracy (best performing model)
  • XGBoost: ~86% accuracy
  • Decision Tree: ~84% accuracy
  • Logistic Regression: ~71% accuracy

Random Forest achieved the highest predictive accuracy and demonstrated strong generalization across cross-validation tests, making it the recommended model for loan eligibility prediction.

Alliance Prototype Project: Project Management Simulation

This project evaluates the performance of a simulated software development project using project management principles such as resource allocation, schedule planning, cost control, and quality management.

Project Overview

The Alliance Prototype Project simulation was conducted as part of a project management course to analyze how different management strategies impact project outcomes. The project involved managing a simulated development project across multiple runs while balancing constraints such as budget, schedule, and quality defects. Each run allowed the team to refine strategies, improve decision-making, and apply lessons learned from previous attempts.

Project Goals

  • Complete the project within a $50,000 budget
  • Finish within approximately 11–13 weeks
  • Maintain fewer than 12 project defects
  • Optimize resource allocation and scheduling

Project Planning & Management

The project involved planning tasks, allocating team members, and managing dependencies using project management tools such as Gantt charts and network diagrams. Key project phases included:

  • Systems Design
  • Network Design and Implementation
  • Software Implementation
  • Systems Integration
  • Customer Acceptance Testing
  • Product Documentation

The network diagram illustrates how task dependencies impact project execution, showing that Systems Integration could only begin after Software Implementation and Documentation Review were completed. Delays in early tasks caused cascading impacts on the project timeline. :contentReference[oaicite:2]{index=2}

Project Simulation Runs

The project simulation was conducted in three runs to improve performance through iterative learning.

  • Run 1: Initial attempt faced resource shortages, scheduling conflicts, and dependency delays, resulting in budget overruns and schedule slippage.
  • Run 2: Strategy improvements led to better resource allocation and communication, resulting in improved project performance.
  • Run 3: A collaborative team strategy optimized resource usage and project planning, achieving the best results.

Final Results

  • Final project cost: $49,802 (within the $50,000 budget)
  • Project completion: 100%
  • Total defects: 10 (below the target limit of 12)
  • All project tasks completed successfully

The project demonstrated that effective communication, strategic resource allocation, and continuous monitoring of project metrics can significantly improve project outcomes.
